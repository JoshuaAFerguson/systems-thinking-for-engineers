# Curriculum — Systems Thinking for Engineers

A self-directed, learn-in-public curriculum. It starts from the systems-thinking
canon (Meadows, Senge, Forrester, Sterman) and works outward to engineering
applications — both technical systems and the sociotechnical systems we build them in.

## How this curriculum works

- **17 modules across 4 phases + a capstone.** Work them in order; each builds on the last.
- **Every module carries four artifacts:**
  - `README.md` — objectives, key concepts, and assigned readings *(the "what & why")*
  - `exercises.md` — hands-on exercises: draw the loop, build the model, find the archetype
  - `case-study.md` — apply the module's tools to a real engineering system *(you fill in)*
  - `self-check.md` — recall/reflection questions to test whether it stuck
  - `notes.md` — your synthesis, in your own words (teach-it-back) *(you fill in)*
- **Division of labor:** the assignment side (objectives, readings, exercises, self-checks)
  is written for you. The learning side (`notes.md`, `case-study.md`) is left as a template
  with prompts — that's your work, done in public.

## Working a module

1. Read the module `README.md` and the assigned readings.
2. Do the `exercises.md`.
3. Work the `case-study.md` against a real system you know.
4. Answer `self-check.md` without looking back.
5. Write up `notes.md` in your own words. If you can't explain it simply, loop back.
6. Tick the module off in the [progress tracker](README.md#progress).
