# Module 03 — Feedback Loops

**Phase 1 · Foundations**

Stocks and flows tell you *what* accumulates. Feedback loops tell you *why* the flows change.
A feedback loop is a stock's information about itself looping back to adjust its own flows —
and there are only two flavors. Learn to spot both, label their polarity, and you can read
the engine behind almost any system behavior.

## Objectives

By the end of this module you should be able to:

1. Distinguish a **balancing (negative) loop** — goal-seeking, stabilizing — from a
   **reinforcing (positive) loop** — amplifying, self-multiplying.
2. Read and draw a **causal loop diagram (CLD)**: variables, causal arrows, `+`/`−` link
   polarity, and loop labels (**R** for reinforcing, **B** for balancing).
3. Determine a loop's overall polarity by counting the negative links around it.
4. Recognize the two archetypal loops — the **thermostat** (balancing) and **viral growth**
   (reinforcing) — and find their cousins in your own systems.

## Key concepts

- **Feedback = a loop of causation.** A feedback loop forms when a change in a stock feeds
  back — directly or through a chain — to change the flows that fill or drain that same
  stock. No loop, no feedback; just a one-way chain.
- **Balancing loops (B)** seek a goal and resist change. They compare the stock to a desired
  level and act to close the gap, so they produce stability, equilibrium, and goal-seeking
  behavior. Every regulating mechanism — a thermostat, a body's temperature control, a rate
  limiter, autoscaling toward a target — is a balancing loop.
- **Reinforcing loops (R)** amplify whatever is happening. More leads to more (a *virtuous*
  cycle) or less leads to less (a *vicious* cycle). They drive exponential growth and
  collapse: compound interest, viral adoption, bank runs, burnout spirals, "success to the
  successful."
- **Link polarity.** On a CLD each arrow gets a sign. A `+` link means the two variables move
  the *same* direction (more cause → more effect); a `−` link means *opposite* (more cause →
  less effect). The **loop** is reinforcing if it contains an even number of `−` links
  (including zero) and balancing if it contains an odd number.
- **The thermostat.** The teaching example of a balancing loop: room temperature below the
  setpoint turns the heater on, which raises temperature, which closes the gap, which turns
  the heater off. It hunts toward its goal.
- **Viral growth.** The teaching example of a reinforcing loop: more users → more shares →
  more new users. Left unchecked it explodes — until some balancing loop (saturation, cost,
  fatigue) eventually catches up.
- **Real systems are loops coupled together.** Interesting behavior comes from R and B loops
  interacting, often with delays — which is exactly where Module 04 goes.
