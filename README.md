# Systems Thinking for Engineers

Learning systems thinking in public.

This repo is a self-directed curriculum and working notebook. I'm building a genuine
command of systems thinking — starting from the canon (Meadows, Senge, Forrester,
Sterman) and working outward to real engineering problems, both in the systems we build
and the organizations we build them in.

Everything here is open. The point isn't a polished course; it's a visible trail of the
work: exercises done, real systems analyzed, and concepts explained back in my own words.

## Start here

- **[CURRICULUM.md](CURRICULUM.md)** — the full syllabus: 17 modules across 4 phases + a capstone.

## How each module works

Each module folder is one focused topic with four artifacts:

| File | What it is |
|------|-----------|
| `README.md` | Objectives, key concepts, assigned readings |
| `exercises.md` | Hands-on exercises to actually do |
| `case-study.md` | Apply the tools to a real engineering system *(my work)* |
| `self-check.md` | Questions to test whether it stuck |
| `notes.md` | My synthesis, in my own words *(my work)* |
